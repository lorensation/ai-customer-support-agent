# Quick Start Guide - SaaSify

## Getting Started in 5 Minutes

### Step 1: Create Your Account
1. Visit saasify.com/signup
2. Enter your email and create a password
3. Verify your email (check spam if not received within 2 minutes)
4. Complete your profile (name, company, role)

### Step 2: Create Your First Project
1. Click the **+ New Project** button in the top right
2. Give your project a name (e.g., "Marketing Campaign Q1")
3. Choose a template or start from scratch
4. Add a brief description (optional)
5. Click **Create Project**

### Step 3: Add Tasks
1. In your new project, click **+ Add Task**
2. Enter task name (e.g., "Design landing page")
3. Set:
   - Assignee (yourself or team member)
   - Due date
   - Priority (Low, Medium, High, Urgent)
   - Labels/tags (optional)
4. Press Enter to create more tasks quickly

### Step 4: Invite Your Team
1. Go to **Settings** â†’ **Team** (or click the invite icon)
2. Enter team member email addresses (one per line)
3. Assign roles:
   - **Admin**: Full access to all settings
   - **Member**: Can create and manage projects
   - **Guest**: Limited to assigned projects
4. Click **Send Invitations**

### Step 5: Organize with Views
Switch between different views to match your workflow:
- **List View**: Simple task list with sorting and filtering
- **Board View**: Kanban-style columns (To Do, In Progress, Done)
- **Calendar View**: Tasks displayed by due date
- **Timeline View**: Gantt chart for project planning

## Essential Tips

### Keyboard Shortcuts
- `C`: Create new task
- `P`: Create new project
- `/`: Quick search
- `Ctrl/Cmd + K`: Command palette
- `Escape`: Close dialog

### Quick Task Updates
- Drag tasks between columns in Board view
- Right-click any task for quick actions
- Use bulk actions (select multiple tasks with checkboxes)
- Star important tasks for quick access

### Stay Organized
1. **Use Labels**: Color-code tasks by type (bug, feature, urgent)
2. **Set Dependencies**: Link tasks that depend on each other
3. **Add Subtasks**: Break large tasks into smaller steps
4. **Use Templates**: Save time on recurring projects

### Communication
- **@mention** teammates in comments to notify them
- **Upload files** directly to tasks (drag & drop)
- **Link tasks** to show relationships
- Use **markdown** in descriptions for formatting

## Common Workflows

### Daily Standup
1. Go to your project
2. Filter tasks: "Assigned to me" + "In Progress"
3. Review yesterday's completed tasks
4. Update task statuses
5. Move tasks to "Done" when complete

### Sprint Planning
1. Create a new sprint milestone
2. Estimate tasks (use custom fields for story points)
3. Assign tasks to sprint
4. Track progress with burndown chart (Analytics tab)

### Client Collaboration
1. Invite client as Guest
2. Create a dedicated project
3. Share only relevant tasks
4. Use comments for feedback
5. Set permissions to "View Only" if needed
